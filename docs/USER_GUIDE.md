# CodexMeter 使用说明

CodexMeter 是一个本地运行的 Codex 用量监控工具，主窗口适合查看完整状态，小组件适合长期贴在桌面上观察额度变化。

## 安装与启动

只想直接使用：

1. 打开 [downloads](../downloads/README.md)
2. 下载或双击 `CodexMeter-v0.1.0-win-x64-portable.exe`
3. 按界面提示完成 OAuth 授权

开发模式：

```powershell
npm install
npm run dev
```

构建检查：

```powershell
npm run test
npm run build
```

打包便携版：

```powershell
npm run dist:portable
```

## OAuth 授权

首次使用时连接 Codex OAuth。授权后应用会读取当前账号的额度状态，并在本地保存必要授权信息。

断开连接后，本地保存的授权信息会被清理，应用不会继续刷新额度。

## 额度卡说明

5 小时额度卡用于观察短周期额度，7 天额度卡用于观察周周期额度。

5 小时额度状态：

| 剩余额度 | 状态 |
| --- | --- |
| > 60% | 充足 |
| 30% - 60% | 正常 |
| 20% - 30% | 关注 |
| 10% - 20% | 紧张 |
| 1% - 10% | 预警 |
| 0% | 已耗尽 |

7 天额度状态：

| 剩余额度 | 状态 |
| --- | --- |
| > 60% | 充足 |
| 40% - 60% | 正常 |
| 25% - 40% | 关注 |
| 10% - 25% | 紧张 |
| 1% - 10% | 预警 |
| 0% | 已耗尽 |

## 小组件

打开固定小组件后，可以在桌面上查看：

- 当前套餐，例如 Codex Plus
- 5 小时额度
- 1 周额度
- 剩余比例、已用比例和重置时间

小组件适合配合置顶显示使用。

## 刷新策略

可以在主窗口选择刷新间隔。刷新只读取授权后的用量数据，不会发起模型请求。

如果接口异常、网络不可用或授权失效，界面会保留可识别的状态，重新授权或手动刷新即可恢复。

## 重置卡

右上角菜单会显示可用重置卡数量。应用只展示仍可用的重置卡，已经过期、已兑换或不可用的记录会被过滤。

## 本地与隐私

- OAuth 授权信息只保存在本机
- 不提交 `.env`、Token、缓存和构建产物
- 不读取聊天内容
- 不发起模型请求

## 授权范围

本项目允许学习、个人使用和非商业二次开发，不允许商业使用。商业使用需要先获得作者书面授权。

## 常见问题

### 为什么看不到额度？

先确认已经完成 OAuth 授权，再手动刷新一次。如果仍然没有数据，可能是接口变动或账号侧暂时没有返回额度字段。

### 为什么重置时间和网页不完全一致？

应用读取的是接口返回的周期时间，并按本地时区显示。若官方接口调整字段，显示逻辑也需要同步更新。

### 检查更新怎么做？

当前项目是本地开源版本，暂不强依赖自动更新服务。建议先通过 GitHub Release 发布新版，用户下载新版覆盖安装；后续可以接入 `electron-updater` 和 GitHub Releases。
